# Hybrid Infrastructure Security Simulations — BSc Thesis 🔐
> A BSc final year thesis designing and executing 10 cybersecurity simulations across AWS and Ubuntu, demonstrating real attack scenarios and mitigations across 5 NCSC CAF security principles — built as educational resources for small and medium-sized businesses operating hybrid infrastructures.

---

## 📋 Overview
This project investigates how five core cybersecurity principles operate differently across **AWS Cloud** and **Ubuntu Linux** environments. A fictional 20-person small-t0-medium organisation was built from scratch, deliberately misconfigured to simulate real attack scenarios, then hardened to demonstrate mitigations.

Each simulation follows three stages:
- 🔴 **Setup** — introduce the vulnerability
- 🟠 **Exploitation** — demonstrate the attack
- 🟢 **Solution** — apply the mitigation

---

## 🎯 Security Principles Covered

| Principle | Definition |
|---|---|
| Authentication | Verifying the identity of a user before granting access to a system |
| Privilege Management | Ensuring users only have the level of access required to perform their role |
| Network Security | Protecting the network infrastructure from unauthorised access and malicious traffic |
| Attack Surface Reduction | Minimising the number of entry points into a system to limit exposure to attacks |
| Access Control | Ensuring only authorised users can access, modify or delete data and resources |
